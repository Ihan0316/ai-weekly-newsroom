#소프트웨어 공학

소프트웨어 개발 방법론 중, 요구사항 변경에 유연하게 대응하고 짧은 주기로 반복적인 개발을 통해 점진적으로 결과물을 만들어내는 접근 방식은?

  1. ①폭포수 모델 (Waterfall Model)
  2. ②나선형 모델 (Spiral Model)
  3. ③애자일 모델 (Agile Model)
  4. ④V 모델 (V-Model)

보기를 클릭해 정답을 확인하세요.

정답: ③

애자일 모델은 변화하는 요구사항에 유연하게 대응하기 위해 짧은 개발 주기를 반복하고, 고객과의 지속적인 소통을 통해 점진적으로 소프트웨어를 개발합니다. 반면 폭포수 모델은 각 단계를 순차적으로 진행하며, V 모델은 테스트를 각 개발 단계와 연결하고, 나선형 모델은 위험 관리에 중점을 둡니다.

IT · 개발 · 기획 용어
  • 개발 멱등성 (Idempotence) 동일한 연산을 여러 번 수행하더라도 항상 동일한 결과를 반환하며, 시스템의 상태를 변경하지 않는 특성입니다.
  • 개발 서킷 브레이커 (Circuit Breaker) 분산 시스템에서 장애가 발생한 서비스로의 요청을 일시적으로 차단하여 시스템 전체의 연쇄적인 장애를 방지하는 디자인 패턴입니다.
  • 기획 온보딩 (Onboarding) 새로운 사용자나 직원이 제품이나 서비스, 또는 조직에 성공적으로 적응하도록 돕는 일련의 과정입니다.
