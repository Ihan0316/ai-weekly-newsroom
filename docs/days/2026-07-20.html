정답: ③

애자일 방법론은 요구사항 변경에 유연하게 대응하며, 짧은 주기로 반복적인 개발을 통해 점진적으로 결과물을 만들어냅니다. 고객과의 지속적인 소통과 피드백을 중요하게 여겨 변화하는 환경에 적합합니다. 폭포수 모델은 각 단계가 엄격하게 순차적으로 진행되어 변경에 취약합니다.

03
IT · 개발 · 기획 용어
  • 개발 컨테이너 오케스트레이션 (Container Orchestration) 여러 개의 컨테이너화된 애플리케이션의 배포, 관리, 스케일링, 네트워킹 등을 자동화하는 기술 또는 시스템입니다.
  • IT 백프레셔 (Backpressure) 데이터 처리 시스템에서 생산자의 데이터 생성 속도가 소비자의 처리 속도보다 빠를 때, 생산자의 속도를 조절하여 시스템 과부하를 방지하는 메커니즘입니다.
  • 기획 퍼널 분석 (Funnel Analysis) 사용자가 서비스나 제품에서 특정 목표를 달성하기까지 거치는 단계들을 시각화하여, 각 단계에서의 전환율을 분석하고 병목 구간을 찾아 개선점을 도출하는 방법입니다.
